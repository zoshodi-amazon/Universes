# 03-01-26 — Phase-Based Module Refactor v4.0.0

## Decision

Reorganize Modules/ into 7 canonical semantic phases with Lean 4 as the type system,
JSON as the IO boundary, and Nix as the monadic binding layer.

## Architecture

```
Types/ (Lean 4)           →  default.json  →  Monads/ (Nix)
ADTs + deriving ToJson       IO boundary      builtins.fromJSON → module API calls
compile-time typed           committed file   flake-parts modules
```

## Key Changes from v3.1.0

- Dropped import-tree — IOPipelinePhase explicitly imports 7 phases
- Dropped Nickel — replaced with Lean 4 ADTs
- Dropped scope hierarchy (Fleet/Host/User/Labs) — replaced by 7 semantic phases
- Types/ is a Lean 4 Lake project exporting default.json per phase
- Monads/ reads default.json via builtins.fromJSON — no options blocks
- flake.nix imports only Modules/Monads/IOPipelinePhase
- IOPipelinePhase is the single entry point and only justfile-exposed monad

## Phase Chain

```
Identity → Platform → Network → Services → User → Workspace → Deploy
```

7 phases + 1 Pipeline. Pipeline orchestrates all phases.
