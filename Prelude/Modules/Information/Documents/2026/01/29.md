## Philosophy Notes

### Package → Module Workflow

1. **Identify Capability Space**: What does this package enable?
2. **Map Configuration Surface**:
   - ENV vars → Options
   - CLI flags → Options
   - Config files → Options
3. **Scaffold Module**:
   ```
   <Module>/
   ├── README.md
   ├── default.nix
   ├── Env/           # ENV var aggregation
   ├── Instances/     # Flake exports
   └── Universe/      # Internal capability decomposition
       └── <Feature>/
           ├── Options/   # Type space (possibility)
           └── Bindings/  # Term space (inhabitants)
   ```
4. **Define Options First**: Shape the possibility space
5. **Implement Bindings**: Fill with actual configurations

### Key Insight

> We scaffold the **solution space** first, then mold packages to fit.
> Not vice versa. The capability space is primary; implementations are secondary.

### Adjoint Pairing

```
Options (Free)  ⊣  Bindings (Forgetful)
    ↓                    ↓
Type/Schema         Terms/Instances
Possibility         Actuality
class Foo where     instance Foo Bar where
```
